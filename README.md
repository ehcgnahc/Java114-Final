# 114年 NYUST Java 程式設計期末專案

本專案為雲林科技大學 114 年 Java 程式設計課的期末作業，內容為「銀行系統模擬」

## 環境資訊
- 作業系統：Windows 11
- Java 版本：JDK 21.0.3

## 專案題目：銀行模擬系統(需求摘要)
設計一套盡可能貼近現實的銀行系統，需包含以下主要內容

### 1. 基本類別
- 使用者(User)
- 銀行(Bank)
- 銀行卡(BankCard、DebitCard 或 ATMCard)

### 2. 基本功能
- 使用者可於銀行開戶並獲得卡片
- 可透過銀行或 ATM 完成存款與提款
- 跨行轉帳需收取手續費，各銀行費率可能不同
- 使用者不得直接修改帳戶餘額，所有餘額變動必須透過銀行邏輯處理

### 3. 可擴充功能
- 各銀行可設計不同活動，例如：
  - 點數累積
  - 抽獎
  - 簽帳卡支付回饋
- 支援多種卡片類型(金融卡、簽帳卡等)

### 4. 設計合理性要求
系統邏輯須符合現實。例如：
- user.getCard().setCardBalance(1000) 屬於不合理操作
  因為使用者不應直接改動餘額，僅能透過銀行進行操作。

評分標準如下：
- 功能完整度：70%
- 設計合理性：30%
